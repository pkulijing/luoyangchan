# 洛阳铲个性化功能需求

## 背景

洛阳铲目前是一个纯展示型的文保单位地图浏览工具，缺乏用户互动和个性化功能。为了增强用户粘性和使用体验，需要添加用户系统和个性化功能。

## 需求概述

### 1. 用户登录系统
- 支持邮箱验证码登录（适用于国内外所有用户）
- 支持 Google OAuth 登录（主要面向海外用户）
- 无需复杂的手机号验证，降低用户门槛

### 2. 用户信息维护
- 用户头像（支持上传）
- 昵称
- 个人简介
- 个人主页 URL（基于用户名）

### 3. 标记功能
- 标记"去过"的文保单位
- 标记"想去"的文保单位
- 显示标记统计（多少人去过/想去）

### 4. 成就系统
- 按省份进度成就（如：去过河南省 10/50/100 个）
- 按类别进度成就（如：去过 20 个古遗址）
- 按总数进度成就（如：总共去过 100/500/1000 个）
- 区域完成成就（如：某省/市/县全去过）

## 技术约束

- 使用 Supabase Auth 进行认证
- 使用 Supabase Storage 存储用户头像
- 遵循项目现有技术栈（Next.js、React、TypeScript、Tailwind CSS）
- 考虑中国用户的网络环境（Google OAuth 可选，邮箱验证码必须可用）
