# 需求：地图提供者切换（高德 / 天地图）

## 背景

天地图 WMTS 瓦片服务日限额 10,000 次，每次缩放/拖动加载多个瓦片（vec_w + cva_w 各计一次），开发阶段已触发限流，无法正常测试。

高德 JS API 个人认证开发者免费额度 150 万次/月，且按地图初始化计数（非瓦片请求），实际可用量高出数个数量级。

## 需求

1. 支持在高德 JS API 2.0 和 Leaflet + 天地图之间切换
2. 当前通过环境变量 `NEXT_PUBLIC_MAP_PROVIDER` 控制（`amap` | `tianditu`），默认 `amap`
3. 后续可扩展为管理员配置
4. 两套实现共存，保持相同的 Props 接口
5. 完整复现现有功能：分类图标、文字标签、堆叠标记、访问/想去标记、聚合、地理定位等

## 历史

项目在第 4 轮开发中从高德迁移到了 Leaflet + 天地图（原因是商业授权费用）。git 历史中保留了旧的 AMap 实现（commit `9115af5` 之前），可作为参考起点。
