我们先批量处理这8次集中公布的结果，后面再单独处理几次零星增补。需要对通知内容做文本分析，拿到我们要的结果。以下是几个建议。

1. 我们需要获取的有效信息仍然是 仍然是 编号/名称/时代/地址/分类/批次/收录年份，与heritage_sites_gov.json 的内容类似

```json
  {
    "release_id": "8-1-1",
    "name": "上宅遗址",
    "era": "新石器时代",
    "release_address": "北京市平谷区",
    "category": "古遗址",
    "batch": 8,
    "batch_year": 2019
  },
```

2. 名称/时代/地址 可以直接获取
3. 批次/年份对于每一批是唯一的
4. 分类比较特殊，没有列在每一条中，而是按照分类分别列出的，比如古墓葬下面列了所有古墓葬的条目。所以需要你整理一下。
5. 编号并不是全部都有，我们统一使用简化的规则：批次号+序号，如第8批序号为123的，编号(release_id)保存为 `8-123`，不使用原始的编号列。
