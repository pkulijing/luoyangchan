# 数据富化 - 为搜索系统做数据准备

## 背景

5171 条全国重点文保单位记录的 `description` 字段全部为 null，缺乏结构化的标签和百科链接。这使得：
- 无法实现自然语言搜索（如"山西省所有与唐太宗李世民有关的文物保护遗址"）
- 用户只能通过名称子字符串匹配 + 省份/类别/时代下拉框进行筛选
- 缺少语义信息（相关历史人物、事件、建筑风格等）

## 目标

为每条文保单位记录生成丰富的文本描述和结构化标签，为后续搜索系统（全文搜索 + 语义搜索）奠定数据基础。

## 具体需求

1. **Wikipedia 内容采集**：为 3544 条有 `wikipedia_url` 的记录，从中文 Wikipedia 获取摘要文本
2. **百度百科 URL 及内容采集**：
   - 为全部 5171 条记录获取百度百科 URL（新增 `baike_url` 字段）
   - 为无 Wikipedia 的 1627 条记录，从百度百科获取摘要内容作为 LLM 输入补充
3. **LLM 描述生成**：用 DeepSeek 为每条记录生成 150-300 字描述（填充 `description` 字段）
4. **LLM 标签生成**：用 DeepSeek 为每条记录生成 10-20 个关键词标签（新增 `tags` 字段），覆盖：相关历史人物、历史事件、朝代、建筑风格、宗教信仰、文化类型等
5. **数据库 Schema 更新**：新增 `tags TEXT[]` 和 `baike_url TEXT` 列

## 约束

- LLM 使用 DeepSeek（已有配置和密钥）
- 百度百科使用已有的 `BaiduBaikeClient`（AppBuilder API）
- 所有脚本需支持断点续跑（checkpoint/resume）
- 遵循项目现有的 Python 脚本模式（round 目录、CLI 参数、分批处理）
