# 第11轮：前端优化

## 背景

用户在地图上浏览文保单位时，会经过筛选、缩放、平移找到感兴趣的区域和站点。当前点击标记后的"查看详情"会跳转到 `/site/[releaseId]` 页面，导致：

1. 地图视角（zoom、center）完全丢失
2. 筛选条件重置
3. 点击浏览器返回后回到全国默认视角，需重新操作

这严重影响了"探索式浏览"的体验。

## 需求

### 需求 1：缩放级别文字标签（已完成）

当地图放大到一定程度（zoom >= 10，可调整）时，将标记从圆点+杆切换为文字标签（显示站点名称），便于识别。

### 需求 2：侧边详情面板

点击地图标记时，在页面右侧滑出详情面板，替代跳转到独立详情页。用户无需离开地图即可查看站点信息，关闭面板后地图状态完全保留。

- 面板内容：站点名称、分类、时代、批次、地址、省份城市、百度百科链接、简介、子记录/父记录等
- 面板内保留"查看完整详情"链接，可跳转到独立页面
- 坐标重合的堆叠标记弹窗中的链接也应打开侧边面板而非跳转

### 需求 3：侧边面板宽度调整

侧边详情面板宽度从固定 400px 改为占据屏幕宽度的 1/3，同时设置最小宽度 320px 防止小屏幕上过窄。

### 需求 4：进入页面自动定位到用户位置

进入地图页面时，使用浏览器 Geolocation API 获取用户当前位置，成功后自动定位到用户位置并设置 zoom=11，便于用户探索附近的文保单位。

- 定位成功：以用户位置为中心，zoom=11
- 定位失败（权限拒绝、超时等）：保持原有行为（适应所有标记边界）
- 注意：Geolocation API 要求安全来源，本地开发需使用 `localhost` 而非 `127.0.0.1`
