# 坐标重合可视化与聚合优化

## 背景

数据库中有约 1015 条数据没有精确坐标，地理编码只返回了所在行政区的中心点。这导致多条数据共用同一个坐标，在地图上叠加显示时只能看到一个标记，无法感知背后有多少条数据，不利于排查和修复。

同时，地图默认聚合半径固定为 60px，在全国视野下标记密集，放大后聚合仍然过于激进，影响浏览体验。

## 需求

1. **坐标重合可视化**：以某种方式把共用同一坐标的多条数据显示出来，方便 debug 这批精度不足的记录。

2. **聚合半径优化**：降低默认聚合强度，并支持在调试阶段手动调节聚合半径；进一步地，将聚合半径与地图缩放级别联动，全国视野时适度聚合，放大到城市级别后完全不聚合。
