# 第四轮数据清洗需求

## 背景

第三轮数据清洗（Round 3）完成后，5150 条记录中仍有以下残留问题：

1. **~120 条低精度地址**：DeepSeek + DuckDuckGo 对偏远遗址查找效果差，地址精度只到县级
2. **`_geocode_method` 标签不准确**：1679 条记录标签为 `tencent_geocode_gemini`，实际来源是 DeepSeek
3. **26 条父记录坐标为空**：多地址拆分后的父记录没有坐标
4. **地址字符串缺少行政层级**：如"山东省滕州市官桥镇北辛村"缺少中间的"枣庄市"
5. **全量数据未做最终质量审计**：可能存在的模糊地址和重复坐标未被系统排查

## 目标

1. 执行第三轮遗留的 3 个 TODO
2. 规范化地址字符串，补全缺失的行政层级
3. 对全量数据做最终质量审计，输出报告供人工决策

## 具体需求

### TODO 1：百度百科补全低精度记录
- 引入百度百科 API（`skills/baidu-baike`）为 ~120 条低精度记录查找精确地址
- 流程：扫描 result 文件识别低精度 → 百度百科查询 → DeepSeek 合成地址 → 腾讯地图 geocoding

### TODO 2：`_geocode_method` 标签规范化
- 将 `tencent_geocode_gemini` 重命名为 `tencent_geocode_deepseek`

### TODO 3：父记录坐标处理
- 计算子记录坐标质心作为父记录坐标

### 新增：地址字符串规范化
- 利用已有的 `province`/`city`/`district` 结构化字段，纯规则拼接补全地址中缺失的行政层级
- 不需要调用外部 API

### 新增：全量质量审计
- 排查模糊/县级地址，输出报告
- 排查重复 GPS 坐标，输出报告
- 报告供人工审阅后决定后续动作
